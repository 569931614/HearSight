# video-browsing

## MODIFIED Requirements

### Requirement: Video Metadata Include Mind Map Availability
The video browsing system SHALL indicate whether mind map data is available for each video.

**Priority:** Low
**Category:** Data Display

#### Scenario: Video list shows mind map indicator
- **Given** the user is viewing the video gallery
- **And** some videos have mind map data and others do not
- **When** video cards are displayed
- **Then** videos with mind maps show a visual indicator (optional enhancement)
- **Note:** This is a future enhancement and not required for initial implementation

#### Scenario: Video details include mind map field
- **Given** a video is selected and details are fetched
- **When** GET /api/qdrant/videos/{video_id}/paragraphs is called
- **Then** the response includes a `has_mind_map: boolean` field (optional)
- **And** the frontend can use this to pre-render or hide the mind map tab
- **Note:** This is an optional optimization and not required for MVP

## ADDED Requirements

### Requirement: Mind Map API Endpoint
The backend SHALL provide an endpoint to retrieve mind map data for videos.

**Priority:** High
**Category:** Backend API

#### Scenario: Fetch mind map data for video
- **Given** a video with ID "abc123" has mind map markdown stored
- **When** a GET request is made to /api/qdrant/videos/abc123/mindmap
- **Then** the response status is 200 OK
- **And** the response body contains:
  ```json
  {
    "video_id": "abc123",
    "mind_map_markdown": "# Video Title\n## Topic 1\n### Subtopic 1.1",
    "generated_at": "2025-12-13T10:30:00Z",
    "version": "1.0"
  }
  ```

#### Scenario: Request mind map for video without data
- **Given** a video with ID "xyz789" does not have mind map data
- **When** a GET request is made to /api/qdrant/videos/xyz789/mindmap
- **Then** the response status is 404 Not Found
- **And** the response body contains:
  ```json
  {
    "detail": "Mind map not found for this video"
  }
  ```

#### Scenario: Request mind map for non-existent video
- **Given** no video exists with ID "invalid123"
- **When** a GET request is made to /api/qdrant/videos/invalid123/mindmap
- **Then** the response status is 404 Not Found
- **And** the response body contains:
  ```json
  {
    "detail": "Video not found"
  }
  ```

### Requirement: Mind Map Storage in Qdrant
The backend SHALL store mind map markdown content in Qdrant alongside video metadata.

**Priority:** High
**Category:** Data Storage

#### Scenario: Store mind map data with video
- **Given** a video is being processed and mind map markdown is generated
- **When** the video metadata is saved to Qdrant
- **Then** the `mind_map_markdown` field is included in the payload
- **And** the field accepts text content up to 10MB
- **And** null or empty string is accepted for videos without mind maps

#### Scenario: Update existing video with mind map
- **Given** a video "abc123" exists without mind map data
- **When** mind map markdown is generated for this video
- **Then** a PUT request to /api/qdrant/videos/abc123/mindmap stores the data
- **And** the Qdrant vector is updated with the new field
- **And** the `mind_map_generated_at` timestamp is recorded

#### Scenario: Query mind map field efficiently
- **Given** multiple videos are stored in Qdrant
- **When** retrieving video details
- **Then** the mind map field can be optionally excluded from the response
- **And** querying only mind map data does not require fetching full video embeddings
- **Note:** This optimization ensures efficient data retrieval

### Requirement: Mind Map PUT Endpoint
The backend SHALL provide an endpoint to store or update mind map data (for future AI integration).

**Priority:** Medium
**Category:** Backend API

#### Scenario: Store new mind map data
- **Given** a video with ID "abc123" exists
- **And** mind map markdown has been generated by AI
- **When** a PUT request is made to /api/qdrant/videos/abc123/mindmap with:
  ```json
  {
    "mind_map_markdown": "# New Mind Map\n## Topic 1",
    "version": "1.0"
  }
  ```
- **Then** the response status is 200 OK
- **And** the mind map is stored in Qdrant
- **And** subsequent GET requests return the new mind map

#### Scenario: Update existing mind map
- **Given** a video "abc123" already has mind map data
- **When** a PUT request updates the mind map with new content
- **Then** the old mind map is replaced with the new one
- **And** the `mind_map_generated_at` timestamp is updated

#### Scenario: Validate mind map content
- **Given** a PUT request with invalid markdown (e.g., exceeds 10MB)
- **When** the backend processes the request
- **Then** the response status is 400 Bad Request
- **And** an error message indicates the validation failure
